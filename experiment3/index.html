Description

For this experiment, I set out to create two distinct procedural worldsâ€”an overworld and a dungeon. I used noise-based generation techniques along with a provided tilemap to differentiate the two environments. By customizing the generation logic and tile placement, I was able to give each world its own visual identity and structure.

Technical

The overworld uses Perlin noise to generate natural terrain variation like grass, trees, and water, with animated lighting effects that simulate life over time. The dungeon also utilizes noise for the shadows and lighting effect. It also features a snake-like path generation algorithm with randomly placed rooms connected to the paths and uses bitmasking for smooth tile transitions. Both worlds are built on a grid system, support reseeding for randomness, and map character codes to tiles using a shared rendering function..

Reflection

I think the largest drawback I had with this assingment was getting the coordinates for the tilemap and autotailing. It truly felt like an intricate process as I spent so much time trying to meticulously match the coordinates of the assets with the placement of how I wanted my worlds.
